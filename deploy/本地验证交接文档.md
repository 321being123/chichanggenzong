## 五、本地起 PG 的三种方案（接手人三选一）

### 方案 A（推荐，最干净，免管理员 / 免装 / 不注册服务）：EDB 二进制 zip
> 已验证 zip 可达（HTTP 200）：`postgresql-16.4-1-windows-x64-binaries.zip`（323MB）。
```bash
# 1. 下载（约 323MB）
mkdir -p ~/pg_install
curl -L -o ~/pg_install/pg16.zip https://get.enterprisedb.com/postgresql/postgresql-16.4-1-windows-x64-binaries.zip

# 2. 解压（本机 unzip 可用）
unzip ~/pg_install/pg16.zip -d ~/pg

# 3. 初始化数据目录（trust 认证免密码，仅限本地验证）
~/pg/bin/initdb.exe -D ~/pgdata --auth=trust -U postgres

# 4. 启动（不注册 Windows 服务）
~/pg/bin/pg_ctl.exe -D ~/pgdata -l ~/pglog start

# 5. 建库（initSchema 只建表不建库，必须先建 portfolio）
~/pg/bin/createdb.exe -U postgres portfolio
```
- 停止：`~/pg/bin/pg_ctl.exe -D ~/pgdata stop`
- 清理：直接删 `~/pg` 与 `~/pgdata` 即可，零系统残留。

### 方案 B：本机用管理员装官方 installer（适合长期开发）
- 管理员终端运行 `winget install PostgreSQL.PostgreSQL`，或双击官方 `.exe`；
- 设超级用户密码，建库 `portfolio`；
- 改 `deploy/.env`：`PGHOST/PGPORT/PGUSER/PGPASSWORD/PGDATABASE`。

### 方案 C（最快）：用已有 / 云上 PG 连接串
- 直接拿到 `DATABASE_URL` 或 `PG*` 变量（如腾讯云 PostgreSQL）；
- 填进 `deploy/.env` 或环境变量，跳过"起实例"这一步。

3. `db.js` 默认库为 `portfolio`，**勿把 `PGDATABASE` 设成 `postgres`**，否则表建错库。

## 九、已知风险 / 注意
1. 方案 A 用 `trust` 认证**仅限本机验证**，生产必须改密码 + SSL（`PGSSL=true`）。
2. `initSchema` 只建表、不建库——连库前务必先 `CREATE DATABASE portfolio`，否则连接即报错。
3. 若本机缺 VC++ 运行库（本机已确认齐全，通常不需），先装 `vc_redist.x64.exe`。
4. 云上 PG 可能限制 `CREATE EXTENSION` 权限；本改造 `initSchema` 未用到扩展（密码用 Node `crypto`），无此顾虑。
