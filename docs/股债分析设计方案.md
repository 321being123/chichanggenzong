# 股债分析设计方案

## 目标

- 一级导航“股票分析”改为“股债分析”。
- 股票和可转债统一输入六位代码，自动识别后展示分析结果。
- 可转债结果全部采用表格，不使用指标卡片。
- 数据首次全量入库，后续按交易日、公告日、报告期增量更新；页面只读数据库快照。
- 外部数据失败时保留最后一份成功快照，不用空结果覆盖。

## 数据分层

1. `ops.raw_records` 保存外部原始数据和内容哈希。
2. `core`、`market`、`fundamental` 保存标准化证券、行情、财务和可转债条款。
3. `analytics.metric_values` 保存带公式版本的计算指标。
4. `analytics.analysis_snapshots` 保存页面最终快照。
5. `ops.sync_cursors` 保存各数据集增量游标，`ops.data_quality_issues` 保存缺失、过期和权限问题。

现有 `core.instruments`、`market.daily_bars`、`market.latest_quotes`、`market.daily_valuations`、财务表和分析快照继续复用。仅新增可转债特有表：基本资料、条款、票息、评级、转股价历史、不下修历史、基金持仓和每日触发计数。

## 同步策略

- 首次启动且没有可转债资料时，全量同步存续可转债基础资料和最近交易日行情。
- 每日收盘后全量读取当日市场分区，以主键幂等写入；基础资料按内容更新。
- 评级、转股价历史等按单只转债增量补充，接口无权限时明确记录，不伪造数据。
- 用户查询已缓存且未过期的数据时直接返回；手动刷新只刷新当前代码。
- 实时转债价与正股价必须同时可用才计算盘中转股价值和溢价率，否则整组使用同一交易日收盘数据。

## 可转债页面

依次展示：行情概览、可转债基本信息、触发进度、募资用途与条款、转股价调整历史、不下修历史、可转债安全性、债券基本信息与票息、期权价值、正股相关、数据来源与质量。

字段覆盖：正股价、转股价、转股价值、转股溢价率、强赎/下修/回售触发价及计数、基金持仓、到期时间、剩余年限和规模、最快回售日期及收益率、转债/总市值、募资用途、各项条款、转股价历史、不下修历史、安全性三指标、信用评级、纯债价值、债底溢价率、票息、到期收益率、担保、期权价值、理论价值、理论偏离度、正股PE/PB/波动率和资产负债率。

所有缺失值区分为“暂无数据”“不适用”“权限不足”“数据过期”，不得转换成零。

## 计算口径

- 转股价值：`正股价 / 转股价 × 100`。
- 转股溢价率：`转债价 / 转股价值 - 1`。
- 双低值：`转债价 + 转股溢价率百分点`。
- 转债/总市值：`剩余规模 / 正股总市值`。
- 期权价值：`转债价 - 纯债价值`。
- 正股年化波动率：最近最多250个有效交易日对数收益率标准差乘以 `sqrt(250)`。
- 安全性直接复用可转债安全性快照，不另建一套评级规则。

条款触发价只按该转债实际条款解析；无法可靠识别时显示暂无数据。正式触发计数使用收盘价和交易日，盘中行情不写入正式计数。

## 数据来源

- 腾讯行情：转债和正股盘中价格。
- Tushare `cb_basic`：基本资料、票息说明、转股价、条款、担保和信用等级。
- Tushare `cb_daily`：收盘行情、纯债价值、债底溢价率、转股价值和转股溢价率。
- Tushare `cb_rating`：评级历史和展望。
- Tushare `cb_price_chg`：转股价历史；该接口需要单独权限。
- Tushare `daily`、`daily_basic`：正股历史行情、PE、PB和总市值。
- 现有可转债安全性快照：安全性、利息保障倍数、现金覆盖率、负债/市值。

基金持仓、不下修公告、预期回售到账日、理论期权模型等没有可靠数据时先保留结构并显示数据状态，后续接入正式来源后增量填充。
