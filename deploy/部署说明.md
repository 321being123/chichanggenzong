# 持仓管理系统 · 腾讯云部署说明

> 适用：把本地运行的持仓管理系统（Node + PostgreSQL + Express）部署到腾讯云 CVM（Linux）。
> 前置：已修改好的 `server.js`（CSRF 读 `ALLOWED_ORIGIN`、启用 `trust proxy`、cookie `secure:auto`）已就位。
> 存储已从 SQLite 改为 PostgreSQL（用户 2026-07-08 决策，计划开放给几十人使用）。

## 一、服务器端准备

1. **买 CVM**：选 Ubuntu 22.04 / 24.04（或 CentOS 7+），地域随意。
2. **装 Node**：服务器 Node 版本必须与本地一致（当前 **22.x**）。
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
   sudo apt-get install -y nodejs
   node -v   # 应显示 v22.x
   ```
3. **装 PostgreSQL + Nginx + pm2**：
   ```bash
   sudo apt-get install -y postgresql nginx
   sudo npm install -g pm2
   ```
4. **建库 + 建专用账号**（不要用默认 postgres 超级用户跑业务）：
   ```bash
   sudo -u postgres psql <<'SQL'
   CREATE DATABASE portfolio;
   CREATE USER portfolio_user WITH PASSWORD '改成强密码';
   GRANT ALL PRIVILEGES ON DATABASE portfolio TO portfolio_user;
   -- 授权 public schema（PostgreSQL 15+ 需单独授权）
   \c portfolio
   GRANT ALL ON SCHEMA public TO portfolio_user;
   SQL
   ```
   > 服务启动时会由 `server/db.js` 的 `initSchema()` 自动建表（users / account_data / positions / trades / nav_history / cash_flows），无需手动建表。
   > 若数据库启用 SSL（云数据库常见），在 `.env` 设 `PGSSL=true`，并改用 `DATABASE_URL` 连接串。

## 二、上传代码

**不要**直接把 Windows 上的 `node_modules` 拷上去（原生模块平台不兼容，在服务器上用 `npm install` 重装）。

```bash
# 本地打包（排除 node_modules、data、.env、.git）
cd portfolio-server
tar --exclude=node_modules --exclude=data --exclude='.env' --exclude='.git' -czf ../portfolio.tar.gz .

# 传到服务器（替换为你的公网 IP）
scp ../portfolio.tar.gz root@<服务器公网IP>:/root/

# 服务器解压
ssh root@<服务器公网IP>
mkdir -p /opt/portfolio && tar -xzf /root/portfolio.tar.gz -C /opt/portfolio
cd /opt/portfolio
npm install          # pg 是纯 JS 驱动，无需编译原生模块，比 better-sqlite3 更简单
```

> 数据迁移：本系统已彻底脱离 SQLite，无旧 SQLite 文件可迁。全新部署直接启动即可，`server.js` 启动会自动建表（`initSchema()`）。
> 若需从 CSV 批量导入持仓，用 `import_positions.py`（需 `pip install -r requirements.txt`）。

## 三、配置环境变量

> 应用启动时通过 `dotenv` 自动读取项目根目录的 `.env`（无需手动 export），填好即生效。

```bash
cd /opt/portfolio
cp deploy/.env.example .env
nano .env            # 填写 SECRET、ALLOWED_ORIGIN、REGISTER_CODE
```

- `SECRET`：`openssl rand -hex 32` 生成一个
- `ALLOWED_ORIGIN`：改成你的域名，如 `myapp.com,www.myapp.com`
- `REGISTER_CODE`：设一个邀请码（公网必设）
- **PostgreSQL 连接**（必填）：在 `.env` 填 `DATABASE_URL=postgres://portfolio_user:密码@localhost:5432/portfolio`，或分项填 `PGHOST`/`PGPORT`/`PGUSER`/`PGPASSWORD`/`PGDATABASE`/`PGSSL`（详见 `deploy/.env.example`）。

## 四、Nginx 反代 + HTTPS

```bash
cp deploy/nginx-portfolio.conf /etc/nginx/conf.d/portfolio.conf
# 把里面的 myapp.com 和证书路径改成你的实际值
nginx -t && systemctl reload nginx
```

证书：腾讯云控制台申请**免费 SSL**（DNSPod 证书），下载 Nginx 版，放到 `/etc/nginx/ssl/myapp.com/`。

## 五、pm2 守护进程

```bash
cd /opt/portfolio
pm2 start deploy/ecosystem.config.js
pm2 save                 # 固化当前进程列表
pm2 startup              # 开机自启（按提示执行它给出的命令）
```

## 六、腾讯云安全组

只放行 `80`、`443`、`22(SSH)`。**不要**对公网开放 `3000` 端口——它只由 Nginx 在服务器内部访问。

## 七、验证

1. 浏览器打开 `https://你的域名/`，应跳转到登录页且地址栏有锁。
2. 注册/登录/刷新行情/保存数据均正常（不再 403）。
3. `pm2 logs portfolio-server` 看日志；`pm2 restart portfolio-server` 重启。

## 八、备份（重要）

数据在 PostgreSQL，用 `pg_dump` 备份：
```bash
# 手动
pg_dump -U portfolio_user portfolio > /backup/portfolio-$(date +%F).sql

# 每日 03:00 自动（cron）
0 3 * * * /usr/bin/pg_dump -U portfolio_user portfolio | gzip > /var/backups/portfolio_$(date +\%F).sql.gz
```
也可用**云硬盘快照**作为兜底。

## 九、时区与子目录部署（已实现）

- **时区（东八区）**：所有业务日期（净值日期、交易/现金流日期、导出文件名）已统一用北京时间，不再受服务器时区影响；K 线起止日也用北京时间。
- **子目录部署（可选）**：前端接口与静态资源已支持放在域名子目录（如 `https://域名/portfolio/`）。
  开启方法（两步）：
  1. 把两个 HTML 里 `<meta name="base-url" content="">` 的 `content` 改为子目录前缀，例如 `content="/portfolio"`（**必须以 `/` 开头、结尾不带 `/`**）。
  2. Nginx 用带 trailing slash 的 location 反代，把前缀剥掉：
     ```nginx
     location /portfolio/ {
         proxy_pass http://127.0.0.1:3000/;   # 结尾的 / 会把 /portfolio 前缀去掉再转发
         proxy_set_header Host $host;
         proxy_set_header X-Real-IP $remote_addr;
         proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
         proxy_set_header X-Forwarded-Proto $scheme;
     }
     ```
  根目录部署（默认）保持 `content=""`，Nginx 仍用根 location 即可，无需改代码。
