## 二、今日完成的功能

### 1. 可转债安全性

- 新增一级导航“转债安全性”，位置在打新日历、持仓管理之后，版本记录之前。
- 数据源：Tushare 提供转债基础资料、日线估值及财务报表；腾讯财经提供转债和正股最新有效价格。
- 生产数据库已生成首份快照：312 只在市、可交易转债。
- 已过滤退市、到期、停止转股、转股期结束及正股退市的转债。
- 表格字段格式：涨跌幅、股息率、转股溢价率使用百分比；价格、转股价值及三项安全指标保留两位小数；所有数据居中。
- PE：优先 `pe_ttm`，其次静态 PE；均缺失时显示“亏损”。
- PB：Tushare PB 缺失时，用 `total_mv * 10000 / total_hldr_eqy_exc_min_int` 补算；负净资产显示负 PB。
- 支持按评级筛选、搜索、排序及 Excel 导出；导出列与网页一致。

三项债权安全指标：

1. 利息保障倍数：`EBIT / 利息字段`，阈值 `>= 7`。
2. 现金覆盖负债：`max((货币资金+交易性金融资产)/流动负债, (货币资金+交易性金融资产)/有息负债)`，阈值 `>= 1`。
3. 负债/市值：`总负债 / 总市值`，阈值 `<= 1.5`。

银行、非银金融不显示数值，显示“行业豁免”。

### 2. 腾讯行情接入与缓存

- 可转债和港股直接走腾讯批量接口，不再先进入股票 Tushare 路由。
- 每批最多 80 个代码，GBK 解码，逐条解析 `v_symbol` 记录。
- 上海转债与深圳转债按明确前缀映射，避免把全部 `12x` 错发上海。
- 价格必须为有限正数才算有效；HTTP 200 不代表行情有效。
- PostgreSQL `market_quote_cache` 按 `(symbol, source)` 缓存，只写入验证成功的数据。
- 上游超时、空数据或格式异常时保留最后一份有效缓存，避免页面价格突然清空。
- `quote_time` 与 `fetched_at` 分开，休市时“最新”是最后成交价，不等于流式实时价格。
- 转股价值、溢价率和双低只在转债与正股价格同步时按实时值重算，否则整体回退到同一交易日口径。

核心文件：`server/services/tencentQuote.js`、`server/routes/market.js`、`server/services/market.js`。

### 3. 首页与登录模式

- `/` 为默认公开首页，未登录不再强制跳转登录页。
- 首页聚合三个模块：持仓管理、打新日历、可转债安全性。
- 未登录：首页持仓卡显示“登录后查看”，打新与转债摘要正常显示。
- 点击右上角头像进入登录页；未登录点击持仓管理也进入登录页。
- 登录成功后通过 `redirect=/?main=holdings` 自动回到持仓管理。
- 个人账户接口仍由 `requireLogin` 保护；公开范围只开放只读打新及转债快照/导出接口。

核心文件：`public/index.html`、`public/js/home-dashboard.js`、`server/middleware/security.js`、`server/routes/ipo.js`、`server/routes/bondSafety.js`。

### 4. 持仓总资产口径

- 总资产按当前持仓逐项计算：`价格 × 数量 + 现金`。
- 港股按当前港币兑人民币汇率折算。
- A 股优先 Tushare `rt_min`，失败时回退最近日线；港股和转债走腾讯。
- 登录后立即刷新，页面常开时每 15 分钟自动刷新；休市时显示最近有效成交/收盘价。

## 三、重要故障修复

### 服务每 10 分钟崩溃

- 原因：`server/middleware/auth.js` 中 `failMap/regIpMap` 是普通对象，却按 `Map` 迭代和调用 `.delete()`。
- 修复：统一改成真正的 `Map`，抽出 `sweepAuthMaps()` 并增加回归测试。
- 旧 PM2 error log 中仍保留上线前的历史堆栈；新进程已加载修复代码。

### 页面出现 HTML 被当 JSON 解析

- 原因通常是接口请求被未登录重定向到 HTML 登录页，前端仍直接调用 `.json()`。
- 当前首页公开接口不再重定向；个人接口仍返回明确 401 JSON。
- 新代码调用关键接口时应先检查 `response.ok` 和 `content-type`。
