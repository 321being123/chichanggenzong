# 持仓管理系统 · 腾讯云一键部署交接文档

> 本文档记录 2026-07-08 的部署方案与交接要点。
> 目标：让接手的人（或未来的你）能**不依赖我**完成部署与日常运维。

## 一、本次确定的方案（用户拍板）

| 项目 | 决定 |
|------|------|
| 部署方式 | **一键脚本**：服务器从 GitHub 直接拉代码并自动装环境，用户只需在网页终端粘贴一条命令 |
| 数据库 | **本机 PostgreSQL**（装在 CVM 上，与应用同机，最简单）|
| 访问方式 | **公网 IP + HTTP**（域名还没申请下来，先 http://IP 跑起来）|
| 域名/HTTPS | 等域名下来后再加（见第四节）|

## 二、新增的部署物料（已推送 master）

- `deploy/server-init.sh` —— 一键部署脚本（幂等，可重复运行）
- `deploy/nginx-portfolio-http.conf` —— HTTP 版 Nginx 反代（公网 IP 临时用）
- `deploy/nginx-portfolio.conf` —— HTTPS 版（域名下来后换用）
- `deploy/部署说明.md` —— 完整手动部署说明（scp 方式，作为兜底参考）

## 三、部署步骤（给服务器的操作员）

1. 在腾讯云控制台打开 CVM 的**网页终端**（标准登录 / OrcaTerm）。
2. 粘贴下面这一条命令并回车（脚本会自动装 Node22/PostgreSQL/Nginx/pm2、拉代码、建库、生成配置、启动）：

```bash
curl -fsSL https://raw.githubusercontent.com/321being123/chichanggenzong/master/deploy/server-init.sh | sudo bash
```

3. 跑完会打印：
   - 访问地址：`http://<服务器公网IP>/`
   - **注册邀请码**：首次注册必须填（也保存在服务器 `/opt/portfolio/REGISTER_CODE.txt`）
4. **腾讯云安全组**：放行 `80`（和 `22` 方便 SSH），**不要**对公网开 `3000`。

> 兜底：若 CVM 拉 GitHub 很慢/被墙，改用 `部署说明.md` 里的 scp 上传 `portfolio-deploy.tar.gz` 方式手动部署。

## 四、域名下来后切 HTTPS（三步）

1. 腾讯云申请**免费 SSL 证书**（DNSPod），下载 Nginx 版，上传到服务器 `/etc/nginx/ssl/你的域名/`。
2. 替换 Nginx 配置：
   ```bash
   cp /opt/portfolio/deploy/nginx-portfolio.conf /etc/nginx/conf.d/portfolio.conf
   # 编辑该文件，把 myapp.com 和证书路径改成你的实际值
   nginx -t && systemctl reload nginx
   ```
3. 改 `/opt/portfolio/.env` 里的 `ALLOWED_ORIGIN`，从公网 IP 改为你的域名（如 `myapp.com,www.myapp.com`），然后 `pm2 restart portfolio-server`。

## 五、日常运维命令（在服务器上）

```bash
pm2 logs portfolio-server     # 看日志
pm2 restart portfolio-server  # 重启
pm2 status                    # 进程状态
cd /opt/portfolio && git pull && npm install && pm2 restart portfolio-server  # 升代码
```

## 六、数据备份（重要）

数据全在 PostgreSQL，定期 `pg_dump`：
```bash
pg_dump -U portfolio_user portfolio > /backup/portfolio-$(date +%F).sql
# 或每日凌晨自动（cron）
0 3 * * * /usr/bin/pg_dump -U portfolio_user portfolio | gzip > /var/backups/portfolio_$(date +\%F).sql.gz
```
也可用**云硬盘快照**兜底。

## 七、注意事项

- **全新库、数据为空**：本系统已脱离 SQLite，云端是空库，首次使用需重新录入持仓或用 `import_positions.py` 从 CSV 导入。
- **邀请码**：公网部署已强制设 `REGISTER_CODE`，新用户注册必须填；忘记可看 `/opt/portfolio/REGISTER_CODE.txt` 或改 `.env`。
- **改代码后推送**：本项目 push 用保存在本机（Operator 侧）的 GitHub Token，无需再向对话里发密钥。若推送报权限错，说明 token 已失效/被撤销，需重新生成。
- **不要手改库**：`positions`/`trades`/`cash_flows` 才是真数据源；改库请走 API 或 `enter_trade.js`，`cash` 由系统按"期初本金+现金流+交易净额"自动重算。

## 八、回滚 / 重装

- 重跑一键脚本即可（幂等，会复用已存在的库、账号、.env）。
- 彻底重来：删 `/opt/portfolio`、删库 `DROP DATABASE portfolio;`、删 Nginx 配置后重跑。
